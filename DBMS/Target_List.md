# Database Management Systems Implementation Lab

## Phase 1 — Database Design Foundations

1. **Implementation of Entity Relationship (ER) modeling for real-world systems such as library management, hospital management, or e-commerce systems.**

2. **Implementation of ER-to-Relational Schema conversion including entity mapping, relationship mapping, and constraint definition.**

3. **Implementation of normalization workflows including decomposition into 1NF, 2NF, 3NF, and BCNF.**

4. **Implementation of functional dependency analysis and candidate key identification for schema design.**

---

## Phase 2 — Core SQL and Relational Operations

5. **Implementation of Data Definition Language (DDL) operations including CREATE, ALTER, DROP, and schema constraints.**

6. **Implementation of Data Manipulation Language (DML) operations including INSERT, UPDATE, DELETE, and transactional modifications.**

7. **Implementation of SQL query operations using logical, comparison, arithmetic, and conditional operators.**

8. **Implementation of aggregate, string, numeric, date, and group-based query functions.**

9. **Implementation of relational algebra operations including SELECT, PROJECT, UNION, INTERSECTION, DIFFERENCE, JOIN, and DIVISION using SQL equivalents.**

---

## Phase 3 — Query Processing and Data Retrieval

10. **Implementation of multi-table data retrieval using inner joins, equi-joins, non-equi joins, self joins, and outer joins.**

11. **Implementation of nested queries, correlated subqueries, and derived table queries.**

12. **Implementation of Common Table Expressions (CTEs) and recursive query workflows.**

13. **Implementation of query optimization experiments comparing equivalent SQL formulations.**

---

## Phase 4 — Transactions and Concurrency Control

14. **Implementation of transaction workflows using COMMIT, ROLLBACK, SAVEPOINT, and CHECKPOINT operations.**

15. **Implementation of ACID property demonstrations through transactional experiments.**

16. **Implementation of concurrency anomaly simulations including dirty reads, non-repeatable reads, phantom reads, and lost updates.**

17. **Implementation of locking protocols and deadlock simulation experiments.**

---

## Phase 5 — Procedural Database Programming

18. **Implementation of stored procedures using PL/SQL / PostgreSQL procedural language.**

19. **Implementation of triggers for validation, auditing, and automatic event-driven database actions.**

20. **Implementation of cursors for row-wise procedural database processing.**

21. **Implementation of assertions and constraint enforcement mechanisms.**

---

## Phase 6 — Views, Security, and Access Control

22. **Implementation of database views for abstraction, restricted access, and query simplification.**

23. **Implementation of role-based access control using users, privileges, grants, and revocations.**

24. **Implementation of database auditing and permission enforcement workflows.**

---

## Phase 7 — Storage and Indexing Internals

25. **Implementation of indexing experiments using primary indexes, secondary indexes, clustered indexes, and non-clustered indexes.**

26. **Implementation of B+ Tree indexing concepts and search path analysis.**

27. **Implementation of hashing-based indexing experiments for exact-match retrieval.**

28. **Implementation of query performance benchmarking with and without indexing.**

---

## Phase 8 — Query Optimization and System Internals

29. **Implementation of query execution plan analysis using EXPLAIN / EXPLAIN ANALYZE tools.**

30. **Implementation of join strategy comparison including nested loop join, hash join, and merge join behavior analysis.**

31. **Implementation of buffer/cache performance experiments for database I/O behavior understanding.**

32. **Implementation of write-ahead logging and recovery mechanism concept demonstrations.**

---

## Phase 9 — Database Application Engineering

33. **Implementation of a complete relational database-backed application integrating schema design, constraints, transactions, indexing, and query optimization.**

34. **Implementation of reporting dashboards, forms, and data access interfaces over the designed database system.**
